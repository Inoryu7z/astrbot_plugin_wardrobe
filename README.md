# 👗 Wardrobe · 衣橱图鉴

给 Bot 一座属于自己的衣橱，让穿搭不止于文字描述。

**Wardrobe** 是一个图片衣柜管理插件。  
它支持将图片保存到衣柜库中，通过视觉模型自动分析图片内容并生成精细的结构化标签，  
之后可以用自然语言语义检索，让 Bot 在需要时精准找到合适的参考图。

---

## ✨ 它能做什么

### 📸 智能存图

保存图片时，Wardrobe 会调用视觉模型自动分析图片内容，生成完整的结构化属性标签。

分析维度包括：
- **分类**：自动判断"人物"或"衣服"
- **风格**：从 70+ 种预定义风格中匹配（洛丽塔、JK、汉服、纯欲、暗黑等）
- **服装类型**：从 70+ 种服装类型中选择
- **暴露程度**：保守 / 适度 / 略暴露 / 暴露
- **适用场景**：日常、约会、拍照、居家等 40+ 种场景
- **氛围**：清纯、性感、可爱、优雅等
- **姿势与动作**（人物专属）：站姿、坐姿、躺姿、自拍姿势等
- **构图与景别**（人物专属）：特写、近景、中景、全景等
- **表情**（人物专属）：微笑、Wink、嘟嘴、吐舌等可视化表情
- **动作风格**（人物专属）：优雅端庄、性感撩人、活泼俏皮等
- **颜色主调 / 画面构图 / 背景环境**：自由生成描述

用户提供的描述会被强制写入 `description` 字段，确保即使模型识别困难，用户意图也不会丢失。

### 🔍 语义检索

用户可以用自然语言检索图片，例如：
- "有没有洛丽塔发一张看看"
- "发一张甜美一点的衣服来"

检索流程：
1. 取图模型解析用户意图，生成结构化查询条件
2. 数据库按属性匹配取出候选图片
3. 取图模型从候选中选出最匹配的图片
4. 直接发送给用户

如果没有完全匹配的图片，会推荐最接近的结果。

### 🤖 LLM 工具接口

Wardrobe 注册了两个 LLM 工具，对话模型可以自动调用：

| 工具名 | 说明 |
|--------|------|
| `save_wardrobe_image` | 保存图片到衣柜库 |
| `search_wardrobe_image` | 从衣柜库中搜索图片 |

这意味着用户可以用自然语言触发存图和取图，例如：
- "这张自拍挺好看的你存一下" → 自动调用 `save_wardrobe_image`
- "有没有哥特风的图发一张" → 自动调用 `search_wardrobe_image`

### 🔗 AiImg 双向联动（v1.6.0+）

Wardrobe 支持与 AiImg 插件双向联动：

**方向一：AiImg → 衣橱（自动存图）**
- 当 AiImg 插件通过 `aiimg_generate` 生成自拍图片后，Wardrobe 可自动捕获并保存到衣柜库
- 自动调用视觉模型分析图片属性，归入对应人格目录
- 配置项：`auto_save_aiimg_enabled`（默认关闭）

**方向二：衣橱 → AiImg（参考图）**
- AiImg 生图时可调用 Wardrobe 的 `get_reference_image` 接口获取参考图
- 自动排除当前人格的图库，避免同质化（如人格A生图时只从其他人格图库找参考）
- 返回图片路径 + 描述信息，AiImg 可用描述辅助生成更精准的提示词
- 参考图追加到所有人设参考图后面，顺序可控

## 🎮 可用指令

| 指令 | 权限 | 说明 |
|------|------|------|
| `/存图` | 管理员 | 保存图片到衣柜库（回复图片消息或图片+文字同时发送） |
| `/删图 <图片ID>` | 管理员 | 删除衣柜库中的图片 |
| `/衣柜统计` | 所有人 | 查看衣柜库统计信息 |

## ⚙️ 主要配置项

### 模型配置

| 配置项 | 说明 |
|--------|------|
| `save_provider_id` | 存图主模型（需支持视觉能力） |
| `save_secondary_provider_id` | 存图备用模型（主模型超时/失败时自动切换） |
| `search_provider_id` | 取图主模型（需支持视觉能力） |
| `search_secondary_provider_id` | 取图备用模型 |

### 行为配置

| 配置项 | 说明 |
|--------|------|
| `save_timeout_seconds` | 存图模型超时秒数（默认 60） |
| `search_timeout_seconds` | 取图模型超时秒数（默认 30） |
| `save_feedback_enabled` | 存图后是否向用户发送描述和属性信息（默认关闭） |
| `search_candidate_limit` | 从数据库取出的候选图片数量（默认 20） |
| `search_max_select` | 取图模型最多选择的图片数量（默认 1） |
| `max_image_size_mb` | 图片大小限制 MB（默认 10） |
| `personas` | 人格配置列表，每个人格包含规范名和别名列表 |
| `auto_save_aiimg_enabled` | AiImg 生图自动存图开关（默认关闭），开启后自动存入当前对话人格的图库 |

### 人格配置说明

在「人格配置列表」中添加人格，每个人格包含：
- **人格名称（规范名）**：存入数据库时使用的标准名称，如：星织、雪音
- **别名列表**：LLM 可能使用的其他称呼，如：星、小星、星织酱

配置后，存图/取图时会根据当前对话人格自动筛选图片。

**自动存图**：开启「AiImg 生图自动存图」后，AiImg 生成的图片会自动存入当前对话人格的图库。

---

## 📝 使用说明

1. 存图和取图模型均需支持视觉能力，请在配置中选择合适的模型提供商。
2. 存图时如果模型分析失败，图片仍会保存，但标签和描述为空。
3. 开启 `save_feedback_enabled` 后，存图时会返回图片的完整属性分析结果。
4. 检索图片时，结果会直接发送，无需用户额外确认。
